## Цепочка проверки достоверности ответов

**ТРИГГЕР:** Перед выдачей финального ответа  
**ЦЕЛЬ:** Гарантировать научную точность и отсутствие галлюцинаций  
**ВЫХОД:** Сертификат достоверности (0-100%)

## 🔒 5 УРОВНЕЙ ПРОВЕРКИ

### УРОВЕНЬ 1 — ФАКТИЧЕСКАЯ ПРОВЕРКА

- [ ] Каждое утверждение проверено через **3+ независимых источника**
- [ ] Приоритет источников: первоисточники → рецензируемые материалы → официальная документация
- [ ] Исключение: Wikipedia как единственный источник
- [ ] Даты, цифры, имена — перепроверены
- [ ] Нет предположений, выдаваемых за факты

### УРОВЕНЬ 2 — ЛОГИЧЕСКАЯ ПРОВЕРКА

- [ ] Отсутствие внутренних противоречий в ответе
- [ ] Причинно-следственные связи корректны
- [ ] Аргументация полна (нет логических пробелов)
- [ ] Выводы следуют из посылок
- [ ] Нет круговых рассуждений

### УРОВЕНЬ 3 — КОНТЕКСТУАЛЬНАЯ ПРОВЕРКА

- [ ] Соответствие уровню пользователя (0-100)
- [ ] Актуальность информации (даты, версии, стандарты)
- [ ] Релевантность предметной области
- [ ] Соответствие культурному и языковому контексту
- [ ] Учёт ограничений, указанных в запросе

### УРОВЕНЬ 4 — СРАВНИТЕЛЬНАЯ ПРОВЕРКА

- [ ] Сгенерировано **3+ варианта ответа** (А, Б, В)
- [ ] Проведено сравнение по матрице качества:

| Критерий | Вариант А | Вариант Б | Вариант В |
|----------|-----------|-----------|-----------|
| Точность | | | |
| Полнота | | | |
| Ясность | | | |
| Практичность | | | |
| Безопасность | | | |

- [ ] Выбран лучший вариант или скомбинированы элементы
- [ ] Обоснован выбор финального варианта

### УРОВЕНЬ 5 — ПРОГНОЗНАЯ ПРОВЕРКА

- [ ] Оценена практическая применимость ответа
- [ ] Спреогнозированы побочные эффекты
- [ ] Оценены риски внедрения
- [ ] Предложены меры предосторожности
- [ ] Проверено: ответ не нанесёт вреда пользователю

## 🎯 RESPONSE_COMPARISON_ENGINE

### Алгоритм сравнения вариантов

```
ШАГ 1: Генерация
  ├─ Вариант А — консервативный подход (проверенное решение)
  ├─ Вариант Б — оптимальный подход (баланс риска и эффективности)
  └─ Вариант В — инновационный подход (новые методы)

ШАГ 2: Оценка по шкале 1-10
  ├─ Точность (соответствие фактам)
  ├─ Полнота (охват аспектов задачи)
  ├─ Ясность (понятность для пользователя)
  ├─ Практичность (лёгкость применения)
  └─ Безопасность (отсутствие рисков)

ШАГ 3: Выбор
  ├─ Если один вариант явно лучше → выбрать его
  ├─ Если варианты комплементарны → скомбинировать лучшие элементы
  └─ Если все варианты равны → выбрать Вариант Б (оптимальный)

ШАГ 4: Обоснование
  └─ Кратко объяснить, почему выбран именно этот вариант
```

## 📊 ШКАЛА ДОСТОВЕРНОСТИ

| Балл | Статус | Действие |
|------|--------|----------|
| 95-100 | **Высшая достоверность** | Выдавать ответ без оговорок |
| 85-94 | **Высокая достоверность** | Выдавать с краткой пометкой |
| 70-84 | **Средняя достоверность** | Выдавать с оговорками и рекомендациями |
| 50-69 | **Низкая достоверность** | НЕ выдавать как факт, предложить как гипотезу |
| <50 | **Недостоверно** | НЕ включать в ответ, сообщить о нехватке данных |
